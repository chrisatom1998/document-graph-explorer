# Deployment

Document Graph Explorer is a static Vite app. Build artifacts contain all app
code, workers, model files, fonts, and demo assets needed for local-first use.

## Build Products

```bash
npm ci
npm run build
npm run build:airgap
```

- `dist/`: standard web build. Optional Gemini enrichment can call Google's Gemini API only when enabled by the user.
- `dist-airgap/`: sealed air-gapped build. AI surfaces are removed/refused and `scripts/verify-airgap.mjs` fails if an external host is present in the shipped CSP.
- `npm run build:desktop`: wraps `dist/` as a local Electron app.
- `npm run dist:mac`: produces distributable macOS DMG/ZIP artifacts.
- `node scripts/serve.mjs`: serves `dist/`; add `--airgap` to serve `dist-airgap/`.

## Required Headers

Vite dev and preview apply these headers from `vite.config.ts`; production hosts
must send them too:

```http
X-Frame-Options: DENY
X-Content-Type-Options: nosniff
Referrer-Policy: no-referrer
```

Also serve the Content-Security-Policy as an HTTP response header. The build
injects a CSP `<meta>` tag, but `frame-ancestors` only works from a header, which
is why `X-Frame-Options: DENY` remains required.

For the standard build, use the policy generated by `src/security/csp.ts` and add
`frame-ancestors 'none'` when serving it as a header. For the air-gapped build,
the same policy must omit external hosts from `connect-src`; `npm run
build:airgap` verifies that.

COOP and COEP are intentionally omitted. The app uses transferable
`Float32Array`s, not `SharedArrayBuffer`, so cross-origin isolation does not add
value here and can complicate local deployment.

## Nginx Example

```nginx
server {
  listen 443 ssl;
  server_name docs-graph.example.internal;
  root /srv/document-graph-explorer/dist;
  index index.html;

  add_header X-Frame-Options "DENY" always;
  add_header X-Content-Type-Options "nosniff" always;
  add_header Referrer-Policy "no-referrer" always;
  add_header Content-Security-Policy "default-src 'self'; script-src 'self' 'wasm-unsafe-eval' blob:; style-src 'self' 'unsafe-inline'; img-src 'self' data: blob:; font-src 'self' data:; connect-src 'self' blob: https://generativelanguage.googleapis.com https://openrouter.ai; worker-src 'self' blob:; object-src 'none'; base-uri 'self'; form-action 'none'; frame-ancestors 'none'" always;

  location / {
    try_files $uri $uri/ /index.html;
  }
}
```

For `dist-airgap/`, remove both `https://generativelanguage.googleapis.com` and
`https://openrouter.ai` from `connect-src`.

## Checklist

- Run `npm run lint`, `npm run typecheck`, `npm test`, `npm run build`, and `npm run build:airgap`.
- Confirm the deployment serves the required headers on `index.html` and assets.
- Confirm `dist-airgap/index.html` contains no external host in `connect-src`.
- Serve model files from the same origin; do not rewrite `public/models/` to a CDN.
- Keep generated PDFs out of Git unless the release explicitly needs them.
